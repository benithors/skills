# Align

Use this skill to create shared understanding before committing to a plan.

## Workflow

1. Summarize the current understanding of the request.
2. Identify unknowns, assumptions, constraints, and stakeholders.
3. Define success criteria and non-goals.
4. Surface risks, dependencies, and decisions that need user input.
5. Ask one focused question at a time when clarification is required.
6. End with an alignment summary that can feed into planning.

## Rules

- Do not jump into implementation.
- Distinguish facts from assumptions.
- Prefer crisp definitions over vague agreement.
- Call out conflicting requirements directly.
- Keep questions focused and decision-oriented.

## Output Format

```md
## Current understanding
[Concise summary]

## Goals
- ...

## Non-goals
- ...

## Constraints
- ...

## Assumptions
- ...

## Open decisions
1. ...

## Recommended next step
[Clarify / brainstorm / plan / execute]
```
